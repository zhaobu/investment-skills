# 六维捕手 · 通用选股六维评分筛选器

## Overview

本 Skill 基于一套6维评分框架，从全A股市场筛选值得持有3-5年的核心标的，**可适配任何行业**——AI、新能源、消费、医药、半导体、传统制造等。核心原则：**研究结论要经得起3年后回看**。

**适用场景**：
- "帮我用6维标准筛选值得长期持有的标的"
- "看看XX公司适不适合长期持有"
- "帮我评估一下XX的6维评分"
- "筛选XX板块里PE<30、ROE>15%、现金流健康的公司"
- "全市场扫一圈，找出估值合理的行业龙头"

**不适用场景**：
- 短线交易、技术分析、日内操作
- 无数据来源的纯概念炒作
- 港股/美股（仅支持A股，如需支持需明确标注范围）

---

## 分析流程概览

```
Step 1: 理解用户需求 → 确定筛选范围（全市场/指定板块/指定标的）
Step 2: 排雷预筛选 → 自动淘汰主题相关性不足/估值泡沫/概念炒作标的
Step 3: 数据采集 → 批量拉取6维评分所需数据
Step 4: 6维逐项评分 → 按框架打分
Step 5: 交叉验证 → 财务数据/一致预期/机构评级/资金面
Step 6: 输出报告 → 按标准格式输出每只标的的分析报告
```

---

## Step 1: 理解用户需求

首先明确用户的筛选范围和需求类型：

**需求类型A：全市场扫射** → "帮我筛选值得长期持有的标的"
- 使用 `westock-tool filter` 批量初筛候选池
- 初筛条件：`intersect([PE_TTM < 50, ROETTM > 10, TotalMV > 10000000000])`
- 从结果中取 Top 20-30 只，按板块分类后进一步分析

**需求类型B：指定板块深挖** → "帮我看看光伏板块"
- 使用 `westock-data search "<关键词>" --sector` 找到板块代码
- 使用 `westock-data sector <代码>` 获取成分股
- 对成分股逐一6维评分排序

**需求类型C：单标的全维度评估** → "帮我6维评分一下贵州茅台"
- 直接对该标的进行完整6维评分

**需求类型D：自定义筛选条件** → "帮我筛PE<30、ROE>20%、现金流好的"
- 用 `westock-tool filter` 自定义表达式

## Step 2: 排雷预筛选（自动淘汰）

以下类型的标的**无需深入6维评分，直接淘汰**，并在结果中注明淘汰原因：

| 淘汰类型 | 判断标准 | 检查方式 |
|:---|:---|:---|
| **主题相关性不足** | 核心业务与用户关注的主题/行业无关，或收入占比 < 10% | `neodata` 查询主营构成 + `westock-data profile` |
| **估值泡沫** | PE_fwd > 150 或 PE_TTM > 200 或 亏损且无改善预期 | `westock-data quote` → `pe_fwd` / `pe_ratio` |
| **炒作过度** | YTD涨幅 > 150% 且 净利增速 < 50% | `westock-data quote` → `chg_ytd` + `consensus` → `netProfitYoy` |
| **纸面利润** | 经营性现金流/净利润 < 50% （连续3年） | `westock-data finance` 计算最近3年比值 |
| **强周期陷阱** | PE极低但行业处于盈利高点，ROE>30%但不可持续 | 查看毛利率趋势 + 行业供需周期判断 |
| **概念太早期** | 核心产品未量产/无收入验证 | `westock-data report` 研报中查找产品阶段 |

**排雷流程**：
1. 先抓 `quote` 数据检查 PE_fwd、YTD
2. 再抓 `finance` 数据检查现金流/净利润
3. 最后抓 `neodata` 数据判断主题相关性
4. 任意一项触发淘汰线 → 直接跳过，无需后续评分

---

## Step 3: 数据采集

通过排雷的标的，进入完整数据采集流程。每次分析至少拉取以下数据：

### 3.1 基础行情数据
```bash
westock-data quote <代码>
```
取数：`pe_fwd`, `pe_ratio`, `pb_ratio`, `ps_ttm`, `chg_ytd`, `total_market_cap`, `dividend_ratio_ttm`

### 3.2 一致预期数据
```bash
westock-data consensus <代码>
```
取数：`revenue`(2026E/2027E/2028E), `netProfit`(2026E/2027E/2028E), `pe`(各年), `netProfitYoy`(各年), `targetPrice`

### 3.3 财务报表数据
```bash
westock-data finance <代码> --type lrb --num 4    # 利润表
westock-data finance <代码> --type xjll --num 4   # 现金流量表
westock-data finance <代码> --type zcfz --num 4   # 资产负债表
```
关键字段：`OperatingRevenue`, `OperatingCost` → 毛利率；`NPParentCompanyOwners` → 归母净利；`NetOperateCashFlow` → 经营现金流；`AccountReceivable`, `Inventory`

### 3.4 主营构成 & 主题相关性判断
```bash
neodata 查询: "[公司名]主营业务收入构成"
neodata 查询: "[公司名]前五大客户"
neodata 查询: "[公司名] [行业关键词]相关业务收入占比"
```
根据用户关注的行业主题（如新能源、AI、消费、医药等），判断公司核心业务是否与该主题强相关。

### 3.5 行业地位 & 竞争格局
```bash
westock-data report <代码> --limit 5   # 查看研报标题判断地位
neodata 查询: "[公司名] 市占率 [行业名]"
neodata 查询: "[行业名] 竞争格局 龙头"
```

### 3.6 机构评级
```bash
westock-data rating <代码>
```
取数：`rating`（买入/增持/中性/减持）, `targetPrice`（目标价）

## Step 4: 6维逐项评分

### 维度①：稀缺性（权重★★★★）

| 评分 | 标准 | 检查方法 |
|:---|:---|:---|
| 10分 | 全球仅1-2家能做 | 研报确认 + 板块成分股 <3 家同细分领域 |
| 7-9分 | 全球3-5家/国内仅2-3家 | 研报确认 + 有技术/认证壁垒 |
| 4-6分 | 同类5-10家 | 有一定壁垒但竞争加剧 |
| 1-3分 | 大宗同质化/谁都能做 | 无护城河，毛利率持续下降 |

**筛选规则**：✅ 通过≥7分 | ❌ 淘汰<4分

**数据获取**：
1. `westock-data sector <板块代码>` → 统计成分股数量
2. `neodata` 查询行业研报 → 判断细分领域真实竞争者数量
3. `westock-data finance --type lrb` → 查看毛利率趋势（3年持续提升→壁垒强）

---

### 维度②：龙头地位（权重★★★★）

| 评分 | 标准 | 检查方法 |
|:---|:---|:---|
| 10分 | 全球#1，市占率>25% | 研报确认 + 全球龙头 |
| 7-9分 | 国内#1或全球#2-3 | 研报确认 + 国内龙头 |
| 4-6分 | 国内Top3但非龙头 | 研报未明确称"龙头" |
| 1-3分 | 无明确龙头地位 | 排名靠后 |

**筛选规则**：✅ 通过≥7分 | ❌ 淘汰<4分

**数据获取**：
1. `westock-data report <代码> --limit 5` → 提取研报标题中"龙头""领先""第一"等判断
2. `neodata` 查询 `--query "[公司名] 市占率 [行业名]"` → 提取具体百分比
3. `westock-data quote <代码>` → 对比同业市值

---

### 维度③：3-5年确定性（权重★★★★）

| 评分 | 标准 | 检查方法 |
|:---|:---|:---|
| 10分 | 不可绕过的物理/商业瓶颈 | 行业没有它就转不动 |
| 7-9分 | 高概率持续增长，有明确驱动力 | 下游需求确定性强（政策/技术/消费升级） |
| 4-6分 | 有增长但存在替代路径 | 技术路线不确定/竞争格局变化 |
| 1-3分 | 概念性/周期性强 | 远期不确定依赖主题炒作 |

**筛选规则**：✅ 通过≥7分 | ❌ 淘汰<4分

**数据获取**：
1. `westock-data consensus <代码>` → 查看未来3年净利增速（CAGR>20%→确定性高）
2. `neodata` 查询行业研报 → 判断该行业的需求驱动是否具有长期性
3. 检查是否有替代技术/产品路线（颠覆性技术风险）

---

### 维度④：估值未透支（权重★★★★）

| 评分 | 标准 | 检查方法 |
|:---|:---|:---|
| 10分 | PE_fwd < 30 且 2028E PE < 20 | 极度低估，安全边际极高 |
| 7-9分 | PE_fwd 30-50 且 2028E PE < 25 | 估值合理，增长可消化 |
| 4-6分 | PE_fwd 50-80 或 2028E PE 25-40 | 估值偏高但尚可接受 |
| 1-3分 | PE_fwd > 80 或 2028E PE > 40 或亏损 | 估值严重透支 |

**筛选规则**：✅ 通过≥7分 | ⚠️ 观察4-6分 | ❌ 淘汰<4分

**数据获取**：
1. `westock-data quote <代码>` → `pe_fwd`
2. `westock-data consensus <代码>` → 取2028年行的 `pe` 值
3. `westock-data quote <代码>` → `chg_ytd`（YTD>100%需警惕）

**估值评分辅助表**（根据PE_fwd快速定位）：

```
PE_fwd < 20  → 10分（极度低估）
PE_fwd 20-30 → 9分
PE_fwd 30-40 → 8分
PE_fwd 40-50 → 7分
PE_fwd 50-60 → 6分
PE_fwd 60-80 → 5分
PE_fwd 80-100 → 4分
PE_fwd > 100 → 3分
亏损 → 1分
```

**最终分数取 PE_fwd 评分和 2028E PE 评分的较低值**（保守原则）。

---

### 维度⑤：主题纯度（权重★★★）

> **什么是主题纯度**：公司核心业务中，与用户关注的投资主题（如AI、新能源、消费、半导体、医药等）直接相关的收入占比及增速。

| 评分 | 标准 | 检查方法 |
|:---|:---|:---|
| 10分 | 主题收入占比>60%，增速>50% | 主营构成中主题相关产品占绝对主力 |
| 7-9分 | 主题收入占比30-60%，增速>30% | 主题是核心驱动力 |
| 4-6分 | 主题收入占比15-30%，增速>20% | 主题是增量但非核心 |
| 1-3分 | 主题收入占比<15% | 贴主题标签，本质不相关 |

**筛选规则**：✅ 通过≥7分 | ⚠️ 观察4-6分 | ❌ 淘汰<4分

**数据获取**：
1. `neodata` 查询 `--query "[公司名]主营业务收入构成" --data-type api` → 获取各产品线收入占比
2. `westock-data profile <代码>` → 查看前5大客户是否属于主题相关领域
3. `westock-data report <代码>` → 查看研报中主题相关表述

**主题纯度快速对照表**（根据用户关注的行业动态调整评分标准）：

| 投资主题 | 主题相关业务定义 | 典型高分标的 | 判断线索 |
|:---|:---|:---|:---|
| 🔴 **AI/算力** | AI芯片/光模块/服务器/算力租赁 | 中际旭创、海光信息 | 前5大客户=英伟达/谷歌/华为/字节 |
| 🟢 **新能源** | 光伏/锂电池/储能/新能源车 | 宁德时代、比亚迪 | 主营构成中电池/光伏收入占比 |
| 🔵 **半导体** | 芯片设计/制造/封测/设备/材料 | 中芯国际、北方华创 | 晶圆代工/设备收入占比 |
| 🟡 **消费/白酒** | 白酒/食品饮料/品牌消费 | 贵州茅台、伊利股份 | 品牌消费品类收入占比 |
| 🟤 **医药** | 创新药/医疗器械/CXO | 恒瑞医药、药明康德 | 研发投入占比+管线收入 |
| ⚪ **高端制造** | 精密零部件/工业母机/机器人 | 汇川技术、绿的谐波 | 核心零部件收入占比 |
| 🟣 **军工** | 航空/航天/舰船/信息化 | 中航沈飞、航发动力 | 军品收入占比 |

**判断流程**（按优先级）：
1. **最佳**：年报/中报中有明确分部数据 → 直接计算主题收入占比
2. **次优**：无分部数据，但前5大客户或核心产品100%属于主题领域 → 主题收入估算为80-100%
3. **再次**：通过研报描述判断产品用途 → 标注 `[估算: 基于产品用途]`
4. **最差**：无任何主题相关业务描述 → 标注 `[不相关: 业务与主题无直接关联]`

> **缺数据处理**：无法获取主题收入占比时，按产品线+客户结构估算，标注 `[估算: 基于主营构成分析]`。

### 主题纯度的赛道差异化调整

> **核心观点**：不同赛道的合理估值水平天然不同，主题纯度的"门槛"应该随赛道调整。

| 赛道类型 | 纯度7分门槛 | 备注 |
|:---|:---|:---|
| **新兴/小众主题**（如AI/算力/人形机器人） | 主题收入占比>40% | 新概念蹭标的多，要求高 |
| **成熟主题**（如新能源/消费/医药） | 主题收入占比>30% | 标准阈值 |
| **复合主题**（如AI+医药/储能+电力） | 主题收入占比>50% | 必须有清晰主业标签 |
| **传统主题**（如白酒/家电） | 主题收入占比>50% | 不容易蹭 |
| **基础设施/工具型**（如电网/有色） | 主题收入占比>60% | 这类本身"主题感"弱，需高纯度才显特别 |

---

### 维度⑥：现金流验证（权重★★★）

| 评分 | 标准 | 检查方法 |
|:---|:---|:---|
| 10分 | 经营性现金流/净利润 > 120% | 利润有超额现金支撑 |
| 7-9分 | 经营性现金流/净利润 80-120% | 利润有现金支撑 |
| 4-6分 | 经营性现金流/净利润 50-80% | 部分利润未收回 |
| 1-3分 | 经营性现金流/净利润 < 50% | 利润严重缺乏现金支撑 |

**筛选规则**：✅ 通过≥7分 | ⚠️ 观察4-6分 | ❌ 淘汰<4分

**数据获取**：
```bash
# 取最近一期完整年报的现金流
westock-data finance <代码> --type xjll --num 4
# 取同一期利润表的净利润
westock-data finance <代码> --type lrb --num 4
```

**计算**：`NetOperateCashFlow(最近年报) / NPParentCompanyOwners(最近年报) × 100%`

**辅助判断**：同时检查应收账款增速 vs 营收增速、存货增速 vs 营收增速。如果应收账款增速连续3年高于营收增速，即使现金流比率达标，也要在风险提示中说明。

---

### 综合评级规则

**加权总分计算**：
```
加权总分 = 维度1得分×4 + 维度2得分×4 + 维度3得分×4 + 维度4得分×4 + 维度5得分×3 + 维度6得分×3
满分 = 10×4 + 10×4 + 10×4 + 10×4 + 10×3 + 10×3 = 190分
```

| 加权总分 | 评级 | 含义 | 操作建议 |
|:---|:---|:---|:---|
| ≥140分 | 🔴 强烈推荐 | 六维全部优秀，安全边际极高 | 核心仓位 |
| 110-139分 | 🟡 值得关注 | 大部分维度优秀，个别需容忍 | 观察仓位 |
| 75-109分 | 🟢 观察等待 | 有逻辑但有明显短板，等更好价格 | 等待回调 |
| <75分 | ⛔ 淘汰 | 不满足基本投资要求 | 不参与 |

**入围硬性门槛**（同时满足才进入评级）：
1. 每个维度的单项得分都不得低于该维度的淘汰线
2. 如果任一维度被标记为 ❌ 淘汰 → 整个标的综合评级最高为 ⛔ 淘汰

## Step 5: 交叉验证

对通过6维评分的标的，额外验证以下数据：

### 5.1 财务数据验证
```
检查最近4期（最近4个半年或最近4个季度）的：
- 营收是否持续增长（同比增速）
- 毛利率是否稳定或提升
- 净利润是否持续增长
- 经营性现金流是否为正
```

### 5.2 一致预期验证
```
- 2026E/2027E/2028E 净利润是否持续增长
- 修正趋势：近3个月盈利预测是否上调（上调→加分，下调→减分）
- 机构覆盖数量：>10家→可信度高
```

### 5.3 机构评级验证
```
- 买入/增持评级占比（>70%→加分）
- 目标价距当前价空间（>20%→加分）
```

### 5.4 资金面验证（可选）
```
- 北向资金近期流向（近1个月净买入）
- 融资余额变化趋势
```
（若 `westock-data` 能拉到资金流向数据则做，否则跳过）

每只标的按以下标准格式输出：

```markdown
## 📊 [股票名称](股票代码) — 6维评分报告

### 评级总览
| 综合评级 | 加权总分 | 操作建议 |
|:---|:---:|:---|
| 🔴/🟡/🟢/⛔ | XX/190分 | [核心仓位/观察仓位/等待回调/不参与] |

### 六维评分明细

| 维度 | 分数 | 判断依据 | 数据来源 |
|:---|:---:|:---|:---|
| ①稀缺性 | 8/10 | 全球仅3-5家能做，XX壁垒极高 | [研报来源] |
| ②龙头地位 | 9/10 | 全球#1，市占率XX% | [研报来源] |
```

## 预期差识别

最大投资机会来自市场定价错误。在评分完成后，主动扫描以下三类预期差：

### 估值错配
```
条件：PE_fwd < 35 且 主题收入占比 > 30%
含义：市场仍按传统行业估值，但公司业务已转向高增长赛道
示例：传统汽车零部件公司转型机器人零部件，但PE仍仅20x
```

### 增速错配
```
条件：近3年营收CAGR > 30% 且 PE_fwd < 40
含义：增速未被充分定价
示例：公司核心业务增速>50%但PE仅30x
```

### 认知错配
```
条件：公司近半年发生业务转型/新产品投产/大客户突破，但估值倍数无明显变化
含义：市场尚未意识到该标的的赛道已切换
示例：传统制造业公司切入AI/新能源供应链，市场尚未重新定价
```

### 财务质量错配（新增）
```
条件：经营现金流/净利润 > 150% 但 PE_fwd < 行业均值
含义：利润含金量被低估，市场只看到了会计利润，没看现金流
示例：重资产行业龙头，折旧多导致会计利润低，但实际现金流入稳健
```

### 减持噪音错配（新增）
```
条件：过去3个月高管/大股东净减持，但同期业绩超预期
含义：减持可能是个人财务需求而非公司基本面问题，被市场误读
示例：实控人减持1%用于个人理财，但同期订单/财报均超预期
```

## 投资优先级原则

当多个标的通过6维评分时，按以下优先级排序：

1. **估值优先**：同等级赛道，选PE最低的
2. **确定性优先**：有物理瓶颈支撑的 > 有商业逻辑但可替代的
3. **现金流优先**：经营性现金流/净利>80%的 > <50%的
4. **主题纯度优先**：主题收入占比>30%的 > <15%的
5. **预期差优先**：存在预期差错配的 > 已被充分定价的

### 同分时的去重规则

当多只标的总分接近（<10分差距）时，使用以下去重规则：

| 优先级 | 规则 | 说明 |
|:---:|:---|:---|
| 1 | 选PE_fwd更低的那只 | 估值更安全 |
| 2 | 选经营现金流/净利更高的那只 | 利润质量更好 |
| 3 | 选YTD涨幅更小的（>0但<20%） | 还没充分定价 |
| 4 | 选机构覆盖家数>10的那只 | 共识强 |
| 5 | 选市值>200亿的那只 | 流动性好 |

**反规则**：如果候选里有"市值<30亿但增长极好"的标的，单独标记为"高弹性高风险"小盘机会，不直接并入主推荐。

## 边界与规则

### 数据规则
- 所有数据必须标注来源（`[quote]` / `[consensus]` / `[finance]` / `[neodata]` / `[report]`）
- 无数据时标注 `[缺失]` 或 `[估算]`，绝不编造
- 数据超过90天标注 `[STALE]`
- 缺数据时按照 `references/data_mapping.md` 中的替代方案处理

### 排雷清单（必须执行）
- 每次分析前先执行排雷预筛选（Step 2）
- 触发任意淘汰线 → 直接跳过，不浪费评分资源
- 淘汰原因必须在最终报告中明确标注

### 不构成投资建议
- 每份报告末尾必须标注：
  > "本报告仅供参考，不构成个人投资建议。所有数据来源于公开市场信息，可能存在延迟。投资有风险，决策需谨慎。"

### 合规原则
- 涉及内幕消息、未公开信息、监管敏感话题时明确拒绝
- 不评价尚未发布的政策或产品
- 不提供个股的短期买卖建议

### 已知数据源限制
| 限制 | 影响 | 应对 |
|:---|:---|:---|
| `westock-data sector --search` 不存在 | 影响板块代码搜索 | 用 `neodata` 替代 |
| `westock-data report <代码>` 不稳定 | 影响研报数量 | 多次尝试或用 `neodata` 查研报 |
| `westock-data consensus` 的 `institutionCnt` 偶尔为0 | 影响机构覆盖判断 | 用 `neodata` 查机构评级补充 |
| 一致预期数据为0时 | 无法做估值评分 | 标记 `[缺失]`，按"用PE_TTM替代"逻辑 |
