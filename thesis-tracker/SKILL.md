# 投资逻辑跟踪器 · Thesis Tracker

## Overview

**核心思想**：买入只是投资决策的第一步，**持有和卖出才是赚钱的关键**。这个 Skill 解决三个核心问题：
1. **记录**：为什么买？核心逻辑是什么？预期持有期多久？
2. **监控**：哪些数据/事件能验证/推翻逻辑？多久看一次？
3. **行动**：什么时候加仓？什么时候减仓？什么时候清仓？

### 典型触发场景
- "我买了XX股票，帮我跟踪一下"
- "我的腾讯多头逻辑还成立吗？"
- "复盘3个月前买的光伏ETF，现在该加仓还是减仓"
- "我持有的股票该止盈还是止损了？"
- "我的投资组合里哪些标的需要重新评估？"
- "复盘一下6个月前推荐的XX逻辑兑现情况"

### 不适用场景
- 还没有持仓（直接用 `six-dimension-hunter` 选股）
- 短线交易（持仓周期<2周，本Skill主要服务中长线3-6月+）
- 港美股（仅支持A股；如需支持需显式标注范围）

---

## 核心流程

```
Step 0: 持仓档案初始化 → 为每只标的建立完整的"投资档案"
  │
Step 1: 买入逻辑记录 → 写下买入的核心理由和预期触发条件
  │
Step 2: 关键监控指标设定 → 决定每个逻辑需要观察什么数据
  │
Step 3: 定期复盘（周/月/季）→ 对照实际情况检查逻辑兑现度
  │
Step 4: 信号触发识别 → 数据/事件满足预设阈值时生成行动建议
  │
Step 5: 加减仓决策 → 按规则执行
  │
Step 6: 退出/总结 → 卖出时记录退出理由，复盘逻辑正误
```

## Step 1: 买入逻辑记录

**这一步骤是整个 Skill 的灵魂**。没有明确的逻辑，就没有规则化的加减仓决策。

### 逻辑记录三要素

#### 要素1：核心假设（必须可证伪）

```
差的投资逻辑：我觉得它能涨（不可证伪）
好的投资逻辑：储能装机量2026E同比+50%，宁德时代市占率维持35%以上 → 净利增速>20%（可证伪）
```

#### 要素2：触发买入的具体信号

```
❌ 错误：感觉是底部所以买了
✅ 正确：6月10日PE_fwd跌至18x（历史10%分位）+ 北向资金连续3日净流入 + 储能装机数据超预期
```

#### 要素3：什么情况会推翻逻辑

```
❌ 错误：跌了再看看
✅ 正确：核心假设被推翻（如储能装机增速<30%、或公司市占率跌破30%）
```

## Step 2: 关键监控指标设定

为每条核心假设设置**可量化的监控指标**。

### 监控指标设计原则

| 原则 | 说明 |
|:---|:---|
| **可量化** | 必须有数字标准，不能是"看情况" |
| **可获取** | 必须是能查到的数据（财报/公告/官方统计） |
| **有时效** | 必须说明多久更新一次（每周/每月/每季） |
| **有阈值** | 必须说明触发加减仓的具体数值 |

### 监控指标模板

```markdown
## 监控指标（每只标的5-8个）

### 逻辑1：[核心理由1] → 监控指标1
- **数据源**：[westock-data / 财报 / 行业协会]
- **更新频率**：[周/月/季]
- **当前值**：[XX]
- **看多阈值（维持仓位）**：[XX 以上]
- **看空阈值（减仓信号）**：[XX 以下]
- **来源链接**：[获取命令]

### 逻辑2：[核心理由2] → 监控指标2
- ...
```

### 监控指标示例（储能电池龙头）

| 逻辑 | 监控指标 | 数据源 | 看多阈值 | 看空阈值 |
|:---|:---|:---|:---|:---|
| 储能需求高增 | 储能装机量(季度) | 行业协会 | 同比>30% | 同比<10% |
| 公司地位稳固 | 全球动力电池市占率(季) | SNE Research | >30% | <25% |
| 财务质量好 | 经营现金流/净利(年) | 财报 | >100% | <50% |
| 估值合理 | PE_fwd(周) | westock | 15-30x | >50x |
| 短期催化 | 北向资金(日) | westock asfund | 持续净流入 | 连续3日>5亿流出 |

## Step 3: 定期复盘

按设定频率（周/月/季）对照监控指标和实际表现。

### 复盘节奏建议

| 持仓周期 | 复盘频率 | 关键动作 |
|:---|:---|:---|
| 短线1-2周 | 每日盘后 | 快速看股价/重大新闻 |
| 中线1-3月 | 每周1次 | 完整看监控指标 |
| 长线6月+ | 每月1次 | 完整复盘+逻辑兑现度评估 |

### 复盘模板

```markdown
# 📊 [股票名称] — 复盘记录

## 复盘日期：[YYYY-MM-DD]

### 持仓表现
- **当前价**：[XX元]（较建仓：[+/-X%]）
- **持仓市值**：[XX万元]
- **持仓盈亏**：[+/-XX万元（+/-X%）]

### 监控指标兑现度
| 指标 | 设定阈值 | 当前值 | 状态 | 备注 |
|:---|:---|:---|:---|:---|
| 指标1 | 看多>30% | 25% | ⚠️ 接近看空 | 数据[来源] |
| 指标2 | 看多>30% | 32% | ✅ 正常 | 数据[来源] |
| 指标3 | 看空<50% | 65% | ✅ 正常 | 数据[来源] |
| 指标4 | 看空<50% | 30% | ✅ 正常 | 数据[来源] |

### 核心假设验证
- **假设1**：[原假设] → [当前是否仍成立] → [验证数据]
- **假设2**：[原假设] → [当前是否仍成立] → [验证数据]
- **假设3**：[原假设] → [当前是否仍成立] → [验证数据]

### 综合判断
- **逻辑兑现度**：[A.完全兑现 / B.部分兑现 / C.被推翻 / D.不及预期]
- **操作建议**：[继续持有 / 加仓 / 减仓 / 清仓]
- **决策依据**：[具体说明]

### 行动记录
- [ ] 加仓 [数量/价位]
- [ ] 减仓 [数量/价位]
- [ ] 调整监控指标
- [ ] 重新评估逻辑
```

### 逻辑兑现度判定标准

| 兑现度 | 标准 | 操作建议 |
|:---|:---|:---|
| **A. 完全兑现** | 5个监控指标中≥4个在看多阈值内 | 继续持有，坚定信心 |
| **B. 部分兑现** | 5个指标中3-4个在看多 | 继续持有，但密切关注 |
| **C. 不及预期** | 5个指标中2-3个在看多 | 减仓至观察仓位 |
| **D. 被推翻** | 5个指标中<2个在看多 | 清仓 |

## Step 4: 信号触发识别

按预设规则自动识别加减仓信号。

### 加仓信号

| 信号类型 | 触发条件 | 加仓幅度 |
|:---|:---|:---|
| **估值底** | PE_fwd跌至历史20%分位以下 | 加仓20-30% |
| **逻辑加速兑现** | 关键指标超预期+股价回调 | 加仓10-20% |
| **事件催化** | 重大订单/政策落地 | 加仓15-25% |
| **业绩超预期** | 财报披露后业绩>预期20% | 加仓10-20% |
| **资金流入** | 北向/融资连续3日大幅流入 | 加仓10-15% |

### 减仓信号

| 信号类型 | 触发条件 | 减仓幅度 |
|:---|:---|:---|
| **估值顶** | PE_fwd升至历史80%分位以上 | 减仓30-50% |
| **逻辑减速** | 关键指标连续2期走弱 | 减仓20-30% |
| **核心假设被推翻** | 任一核心假设数据证伪 | 减仓50-100% |
| **业绩不及预期** | 财报披露后业绩<预期20% | 减仓30-50% |
| **资金大幅流出** | 北向/融资连续3日大幅流出 | 减仓20-30% |
| **行业进入退潮** | 方向转入退潮期 | 清仓 |

### 清仓信号（强平级别）

| 信号类型 | 触发条件 |
|:---|:---|
| **核心假设彻底证伪** | 业务模式被颠覆（如技术路线失败） |
| **公司基本面恶化** | 财务造假/重大违规/经营停滞 |
| **目标价到位** | 股价达到一致目标价上沿 |
| **更好的投资机会** | 出现六维评分更高的同方向标的 |

### 止损规则

```
初始止损：建仓价×0.85（亏15%无条件减仓一半）
跟踪止损：建仓后最高收盘价×0.85（从最高点回撤15%减仓一半）
硬止损：单只标的浮亏超过总仓位5% → 减仓至1/3
```

## Step 5: 加减仓决策流程

```
收到信号（Step 4 触发）→ 重新评估
   │
   ├─ 加仓信号 → 检查是否还有可用仓位
   │              │
   │              ├─ 是 → 按规则加仓 → 更新档案
   │              └─ 否 → 放弃，记录原因
   │
   ├─ 减仓信号 → 评估核心假设是否被推翻
   │              │
   │              ├─ 是 → 直接清仓
   │              └─ 否 → 按信号强度减仓（30%/50%/100%）
   │
   └─ 清仓信号 → 立即执行
                 │
                 ├─ 记录退出理由
                 ├─ 计算收益
                 └─ 进入复盘流程
```

## 工作规范和协议

### 数据协议
- 每个数字必须标注数据来源：[`quote`] / [`consensus`] / [`finance`] / [`neodata`] / [`report`] / [`WebSearch`]
- 无数据时标注 `[缺失]` 或 `[估算]`，绝不编造
- 数据超过90天标注 `[STALE]`

### 路由优先级
1. **neodata-financial-search**：默认优先，自然语言覆盖广
2. **westock-data**：需结构化数据/批量对比时切换
3. **WebSearch**：前两者均不满足时使用

### 合规底线
- 涉及内幕消息、未公开信息时明确拒绝
- 不评价尚未发布的政策或产品
- 不提供个股短期买卖建议
- 每份报告末尾必须标注免责声明

### 不构成投资建议
> 本报告仅供参考，不构成个人投资建议。

## 关键原则（务必牢记）

1. **买入前先想好怎么卖** —— 没有卖出规则的买入就是赌博
2. **逻辑要可证伪** —— 不能被推翻的"逻辑"等于没有逻辑
3. **量化优先** —— 监控指标必须有数字阈值
4. **承认错误** —— 错了要认，被推翻要清仓
5. **持续迭代** —— 每次退出都是学习机会，更新评分模型和监控指标设计
6. **不和持仓谈恋爱** —— 当核心假设被推翻时，清仓不是"认输"，是"理性纠错"

## 实战场景进阶

### 场景4：补仓 vs 加仓 的区分

> **核心区分**：补仓是降低持仓成本（建仓逻辑没变），加仓是强化建仓逻辑（发现新的买入理由）。

| 维度 | 补仓 | 加仓 |
|:---|:---|:---|
| **触发** | 价格回调 + 核心假设未变 | 出现新催化剂/逻辑加速兑现 |
| **资金** | 总仓位不变，单纯加码 | 总仓位扩大，需要从其他标的挪资金 |
| **风险** | 摊低成本，但放大单标的暴露 | 强化收益，但增加集中度风险 |
| **决策依据** | 价格触发（PE分位/回撤幅度） | 逻辑触发（业绩超预期/事件催化） |

### 场景5：成本基础法 vs 市价法

> 决定减仓/加仓时，是看"相对建仓价的盈亏"还是"相对当前价的预期"？

| 决策类型 | 用法 | 风险 |
|:---|:---|:---|
| **成本基础法** | "我亏了15%所以减一半" | 沉没成本陷阱：亏得越多越不愿卖 |
| **市价法** | "当前PE 50x高于我看空阈值，减仓" | 客观信号驱动，避免情绪化 |

**本 Skill 默认采用市价法**，所有加减仓决策只看"当前价格/数据是否触发信号"，不看"我是赚还是亏"。

### 场景6：组合相关性的隐性风险

> 持仓中5只都是"AI+新能源+科技"概念，看似分散实则高度集中。

| 风险类型 | 表现 | 应对 |
|:---|:---|:---|
| **主题集中** | 多只标的都在"AI"或"储能"主题 | 主题相关标的合计仓位≤30% |
| **行业集中** | 多只标的同属一个行业 | 单一行业≤40% |
| **因子集中** | 多只标的风格类似（如都偏成长） | 大小盘/价值成长/周期防御各占一定比例 |
| **流动性集中** | 多只小盘股，跌时跑不掉 | 单一标的流动性<全组合10% |

**月度组合体检强制检查**：
1. 主题集中度（前3大主题合计仓位）
2. 行业集中度（前3大行业合计仓位）
3. 流动性集中度（低流动性标的合计仓位）
4. 相关性矩阵（标的间相关系数）

### 场景7：黑天鹅事件的应急处理

> 重大利空（财报暴雷/监管处罚/系统性风险）发生时，没有时间走完整流程，需要应急机制。

#### 应急处理流程

```
事件发生 → 5分钟内判断严重程度
   │
   ├─ 严重级（财报造假/退市风险/重大违规）→ 立即清仓（不限价）
   │
   ├─ 中等级（业绩不及预期/管理层变动）→ 减仓50% + 启动复盘
   │
   └─ 轻微级（市场情绪/短期波动）→ 不操作 + 加深监控
```

#### 严重程度判定标准

| 等级 | 触发条件 | 行动 |
|:---|:---|:---|
| **严重** | 退市风险/财务造假确认/重大违规被处罚 | 立即清仓 |
| **中等** | 业绩低于预期30%以上/核心高管离职/重大客户流失 | 减仓50% |
| **轻微** | 季度业绩低于预期/分析师下调评级/行业政策微调 | 不操作+提高监控频率 |

#### 黑天鹅应对原则
1. **不预测，只应对** —— 黑天鹅无法预测，准备好应急机制
2. **保留现金** —— 永远留10-20%现金应对突发
3. **损失可控** —— 单只标的最大损失不超过总仓位5%
4. **不复盘即操作** —— 严重级事件先操作，事后复盘

## 反向思考：常见错误决策模式

> 识别和避免以下7种常见错误。

| 错误模式 | 表现 | 正确做法 |
|:---|:---|:---|
| **沉没成本谬误** | "我亏了20%所以不能卖" | 减仓看的是未来预期，不是历史盈亏 |
| **确认偏误** | "我看到的所有分析都说会涨" | 主动寻找反对意见，列出看空理由 |
| **锚定效应** | "建仓价是XX，所以现在便宜" | 用绝对估值水平（PE/PEG）判断，不是相对建仓价 |
| **损失厌恶** | 盈利10%想跑，亏损20%扛着 | 设统一止盈/止损线，不分情况 |
| **过度交易** | 每天看盘/每周都想调整 | 按周/月/季复盘节奏，不被短期波动驱动 |
| **加仓摊薄** | "跌了所以多买一些拉低成本" | 跌了要重新评估逻辑，不能无脑加仓 |
| **鸵鸟心态** | 亏损后不看不复盘 | 越亏越要严格复盘，要么纠错要么清仓 |
