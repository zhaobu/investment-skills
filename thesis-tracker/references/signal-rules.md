# 加减仓信号详细规则（Step 4）

## 加仓信号

| 信号类型 | 触发条件 | 加仓幅度 | 信号强度 |
|:---|:---|:---:|:---:|
| **估值底** | PE_fwd 跌至历史 20% 分位以下 | 20-30% | 强 |
| **逻辑加速兑现** | 关键指标超预期+股价回调 | 10-20% | 中 |
| **事件催化** | 重大订单/政策落地 | 15-25% | 强 |
| **业绩超预期** | 财报披露后业绩>预期 20% | 10-20% | 强 |
| **资金流入** | 北向/融资连续 3 日大幅流入 | 10-15% | 弱 |

## 减仓信号

| 信号类型 | 触发条件 | 减仓幅度 | 信号强度 |
|:---|:---|:---:|:---:|
| **估值顶** | PE_fwd 升至历史 80% 分位以上 | 30-50% | 强 |
| **逻辑减速** | 关键指标连续 2 期走弱 | 20-30% | 中 |
| **核心假设被推翻** | 任一核心假设数据证伪 | 50-100% | 强 |
| **业绩不及预期** | 财报披露后业绩<预期 20% | 30-50% | 强 |
| **资金大幅流出** | 北向/融资连续 3 日大幅流出 | 20-30% | 中 |
| **行业进入退潮** | 方向转入🍂退潮期 | 100%（清仓） | 强 |

## 清仓信号（强平级别）

| 信号类型 | 触发条件 | 优先级 |
|:---|:---|:---:|
| **核心假设彻底证伪** | 业务模式被颠覆（如技术路线失败） | P0 |
| **公司基本面恶化** | 财务造假/重大违规/经营停滞 | P0 |
| **目标价到位** | 股价达到一致目标价上沿 | P1 |
| **更好的投资机会** | 出现 6 维评分更高的同方向标的 | P2 |

## v1.1 信号冲突处理矩阵

> 现实中常有"估值底 + 核心假设被推翻"同时发生，怎么处理？

| 估值底 | 逻辑加速 | 逻辑减速 | 核心推翻 | 操作建议 |
|:---:|:---:|:---:|:---:|:---|
| ✅ | ✅ | ❌ | ❌ | ➕ 加仓（重仓机会） |
| ✅ | ❌ | ✅ | ❌ | ⏸️ 持有（机会与风险并存） |
| ✅ | ❌ | ❌ | ✅ | ❌ 不加仓（逻辑已破） |
| ❌ | ✅ | ❌ | ❌ | ⏸️ 持有观察 |
| ❌ | ❌ | ✅ | ❌ | ➖ 减仓 30% |
| ❌ | ❌ | ❌ | ✅ | 🚪 清仓 |
| 任意 | 任意 | 任意 | ✅ | 🚪 清仓（核心推翻优先级最高） |

**优先级原则**：
1. **核心假设被推翻 > 一切** → 立即清仓
2. **逻辑验证 > 估值/资金信号** → 看逻辑不看价格
3. **估值信号** 加分/减分时使用 → 不单独触发大幅仓位调整

## 信号强度 vs 仓位调整

| 信号强度 | 仓位调整幅度 |
|:---|:---|
| 强 | ±20-30% |
| 中 | ±10-20% |
| 弱 | ±5-10% |

**最大单次调整**：单只标的单日仓位变化不超过 30%（避免流动性冲击）

## 信号触发后的执行流程

```
信号触发
  │
  ├─ 加仓信号
  │    │
  │    ├─ 检查是否有可用仓位（总仓位 + 现金）
  │    │    ├─ 有 → 按规则加仓 → 更新档案
  │    │    └─ 无 → 记录"信号触发但无可用仓位"
  │    │
  │    └─ 记录决策日志（信号/操作/依据）
  │
  ├─ 减仓信号
  │    │
  │    ├─ 评估是否触发核心假设被推翻
  │    │    ├─ 是 → 直接清仓（不进入减仓流程）
  │    │    └─ 否 → 按信号强度减仓
  │    │
  │    └─ 记录决策日志
  │
  └─ 清仓信号
       │
       └─ 立即执行 → 记录退出理由 → 计算收益 → 进入 Step 6 复盘流程
```
