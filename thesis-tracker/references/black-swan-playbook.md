# 黑天鹅事件应急处理

> 重大利空（财报暴雷/监管处罚/系统性风险）发生时，没有时间走完整流程，需要应急机制。

## 应急处理流程

```
事件发生 → 5 分钟内判断严重程度
   │
   ├─ 严重级（财报造假/退市风险/重大违规）→ 立即清仓（不限价）
   │
   ├─ 中等级（业绩不及预期/管理层变动）→ 减仓 50% + 启动复盘
   │
   └─ 轻微级（市场情绪/短期波动）→ 不操作 + 加深监控
```

## 严重程度判定标准

| 等级 | 触发条件 | 行动 |
|:---|:---|:---|
| **严重** | 退市风险/财务造假确认/重大违规被处罚 | 立即清仓 |
| **中等** | 业绩低于预期 30% 以上/核心高管离职/重大客户流失 | 减仓 50% |
| **轻微** | 季度业绩低于预期/分析师下调评级/行业政策微调 | 不操作+提高监控频率 |

## 黑天鹅类型应对清单

### 1. 公司层面黑天鹅

| 事件 | 严重等级 | 应对 |
|:---|:---:|:---|
| 财务造假被坐实 | 严重 | 立即清仓 |
| 退市风险（如 ST/重大违法） | 严重 | 立即清仓 |
| 重大违规被处罚 | 严重 | 立即清仓 |
| 核心高管被查 | 严重 | 立即清仓 |
| 业绩低于预期 > 30% | 中等 | 减仓 50% |
| 管理层集体离职 | 中等 | 减仓 50% |
| 重大客户流失（>30% 营收） | 中等 | 减仓 50% |
| 业绩低于预期 10-30% | 轻微 | 不操作+提升监控 |
| 分析师下调评级 | 轻微 | 不操作+关注原因 |

### 2. 行业层面黑天鹅

| 事件 | 严重等级 | 应对 |
|:---|:---:|:---|
| 政策重大逆转（如双减之于教培） | 严重 | 评估逻辑 → 大概率清仓 |
| 行业供给侧重大事件（如产能过剩） | 中等 | 减仓 30-50% |
| 替代技术路线突破 | 中等 | 减仓 30-50% |
| 行业增速大幅下台阶 | 轻微 | 提升监控，暂不操作 |

### 3. 系统性黑天鹅

| 事件 | 严重等级 | 应对 |
|:---|:---:|:---|
| 重大金融危机（如 2008/2015 股灾） | 严重 | 减仓至 < 30%，保留现金 |
| 重大地缘冲突 | 中等 | 减仓至 < 50% |
| 美联储超预期加息 | 轻微 | 视持仓 PE 水平调整 |
| 汇率大幅波动 | 轻微 | 视出口占比调整 |

## 黑天鹅应对原则

1. **不预测，只应对** —— 黑天鹅无法预测，准备好应急机制
2. **保留现金** —— 永远留 10-20% 现金应对突发
3. **损失可控** —— 单只标的最大损失不超过总仓位 5%
4. **不复盘即操作** —— 严重级事件先操作，事后复盘
5. **避免恐慌性抛售** —— 区分"事件严重"和"市场恐慌"
6. **事件后 24 小时冷静期** —— 避免立即反手操作

## 黑天鹅后的复盘

事件处理完成后 1 周内必须完成以下复盘：

```markdown
# 🦢 黑天鹅事件复盘 — [股票名称] — [YYYY-MM-DD]

## 事件概要
- **事件类型**：[财报/监管/行业/系统性]
- **严重等级**：[严重/中等/轻微]
- **发生时间**：[YYYY-MM-DD HH:MM]
- **发现途径**：[盘面/公告/新闻]

## 应对执行
- **实际执行操作**：[清仓/减仓/未操作]
- **执行时间**：[事件后 X 小时]
- **操作依据**：[严重等级判定标准]

## 损失评估
- **持仓亏损**：[XX% / XX 元]
- **是否触及止损线**：[是/否]
- **本次事件总损失占组合**：[X%]

## 事件是否被预测到
- [ ] 完全预测到
- [ ] 局部预测到（事件类型对，标的错）
- [ ] 未预测到

## 6 维评分是否仍然有效
- [ ] 完全有效（逻辑未变）
- [ ] 部分有效（部分逻辑被破）
- [ ] 完全失效（重新评分）

## 经验教训
- **做对了什么**：[1-2 条]
- **做错了什么**：[1-2 条]
- **下次如何改进**：[1-2 条]
```

## 黑天鹅预防措施

| 措施 | 说明 |
|:---|:---|
| **仓位分散** | 单只标的 < 20%，单一行业 < 40% |
| **保留现金** | 永远 10-20% 现金 |
| **黑天鹅清单** | 列出每只持仓可能的黑天鹅事件，预先设定应对 |
| **财务排雷** | 定期检查应收账款/存货异常 |
| **政策跟踪** | 关注行业政策变化 |
| **舆情监控** | 关注公司/行业的早期负面信号 |
