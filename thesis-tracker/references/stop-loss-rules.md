# 止损规则

> **v1.1 改进**：原规则一刀切 -15% 忽略了波动率差异，导致高波动票频繁触发或低波动票止损过宽。

## 三层止损防线

### 防线 1：固定止损（基础规则）

| 规则 | 计算方式 | 触发后行动 |
|:---|:---|:---|
| **初始止损** | 建仓价 × 0.85（亏 15%） | 减仓一半 |
| **跟踪止损** | 建仓后最高收盘价 × 0.85（从最高点回撤 15%） | 减仓一半 |
| **硬止损** | 单只标的浮亏超过总仓位 5% | 减仓至 1/3 |

### 防线 2：ATR 止损（v1.1 新增，推荐使用）

> ATR（Average True Range）= 过去 20 日平均真实波幅，反映标的的"日常波动幅度"。

**计算公式**：
```
ATR(20日) = SUM(过去20日TrueRange) / 20
TrueRange = MAX(High-Low, |High-PrevClose|, |Low-PrevClose|)

止损价 = 建仓价 - 2 × ATR(20日)
```

**适用场景**：
- **高波动票**（AI/创新药/小盘股）：建议 ATR 倍数 2.0-2.5（避免频繁触发）
- **中波动票**（新能源/半导体）：建议 ATR 倍数 1.5-2.0
- **低波动票**（白酒/家电/大盘蓝筹）：建议 ATR 倍数 1.0-1.5

**数据获取**：
```bash
# 手动计算
westock-data kline <代码> --period day --limit 20
# 然后用 Python 计算 20 日 ATR
```

**示例**：
- 标的 A（高波动，AI）：建仓价 100 元，ATR(20)=8 元 → 止损价 = 100 - 2×8 = 84 元（-16%）
- 标的 B（低波动，白酒）：建仓价 100 元，ATR(20)=3 元 → 止损价 = 100 - 2×3 = 94 元（-6%）

### 防线 3：逻辑止损（最高优先级）

> **逻辑止损 > 价格止损**

| 触发条件 | 行动 |
|:---|:---|
| 任一核心假设被推翻 | 立即清仓（不限价） |
| 6 维评分从 ≥ 140 跌至 < 75 | 清仓 |
| 行业进入🍂退潮期 | 清仓 |

**核心原则**：当逻辑破了，价格再低也不应该持有。

## 止损规则选择决策树

```
建仓时
  │
  ├─ 标的波动率 ATR(20) / 价格 < 3% → 固定止损（-15%）
  │
  ├─ 标的波动率 3-5% → 混合止损（固定 -15% + ATR 1.5 倍）
  │
  └─ 标的波动率 > 5% → ATR 止损（2 倍）

持仓过程中
  │
  ├─ 逻辑不变 → 跟踪止损
  │
  └─ 逻辑破 → 立即清仓（不限价）
```

## 止损纪律

1. **止损价一旦设定，触及必执行** —— 不要"再看看"
2. **不复盘即操作** —— 触及止损线先减仓，事后复盘
3. **止损后冷静 24 小时** —— 避免立即反手买入
4. **记录止损原因** —— 为后续优化提供数据

## 反规则：避免"过早止损"

以下情况**不**应触发止损：
- 大盘系统性下跌（>5%），但个股逻辑未变 → 不止损，加仓机会
- 短期事件冲击（如分红除权、指数调整）→ 不止损
- 季度财报短期低于预期但年度指引未变 → 持有观察，不止损
